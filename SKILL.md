# qiuyuan-skill · 球员思维蒸馏系统

> 「足球是空间的博弈，谁制造空间、谁利用空间，谁就赢。」

## 核心定位

qiuyuan-skill 不是复制一个球员，而是**提炼他踢球的思维操作系统**。

目标不是得到 8 份分散报告，而是从多维材料中抽出几条稳定的因果链：
- 他如何看空间
- 他如何做技术决策
- 他的身体边界如何限制选择
- 他所处的媒体 / 团队环境如何塑造行为
- 他绝对不会怎么踢、怎么说、怎么应对

**关键区分**：捕捉的是 how they play，不是 what they said。

---

## 工作边界

qiuyuan.skill 只负责**蒸馏与构建**，不直接扮演球员本身。

| 工具 | 作用 | 产物 |
|------|------|------|
| `qiuyuan.skill` | 做调研、提炼框架、生成最终 Skill | `player-perspective.skill/SKILL.md` |
| 生成后的球员 Skill | 在新对话里以球员视角回答问题 | 对话本身 |

蒸馏完成后，要切换到新的对话加载生成的球员 Skill，才能真正进入该球员视角。

## 总流程

```text
Phase 0  入口分流
Phase 1  8 路并行研究
Phase 2  跨维度发现矩阵
Phase 3  提炼与检查
Phase 4  构建最终 Skill
```

原则很简单：

1. 先把材料分维度看清
2. 再把维度之间连起来
3. 最后才写角色化 Skill

没有交叉矩阵，就不要直接提炼；没有提炼确认，就不要直接生成。

## Phase 0 · 入口分流

Skill 激活后，先判断用户是在：
- **直接指定球员**
- **提出模糊战术需求**

### 0A. 明确球员名 → 需求澄清

当用户已经给出球员名时，逐个确认以下信息：

1. 身份：现役 / 退役，若已知先给出判断再请用户确认
2. 聚焦：战术 / 技术 / 精神 / 全面画像
3. 用途：战术顾问 / 球员对比 / 比赛分析 / 其他
4. 本地素材：是否已有采访、字幕、比赛笔记、数据报告

**交互原则**：
- 一次只问一个问题
- 能自己判断的，不让用户重复劳动
- 问完后先汇总，用户明确说“开始蒸馏”再进入 Phase 1

### 0B. 模糊需求 → 候选推荐

如果用户不知道该蒸馏谁，而是在问：
- “我需要一个能内收的前锋模板”
- “这个位置该怎么踢”
- “怎么评价这种类型的中锋”

先不要强行蒸馏，先推荐 2-4 个候选球员。

推荐格式：

```markdown
### 候选：球员名 ⚡已有 Skill / 🆕需要蒸馏
- 核心踢法：
- 适配原因：
- 局限：
```

推荐的重点不是“名气大”，而是**风格与需求是否匹配**。

### subagent 执行要求

每个 subagent 只做自己的维度，不做跨维度总结。

每份报告统一结构：

```markdown
## [维度名]

### 核心发现
- 3-5 条关键发现，每条尽量附来源或场景

### 信息来源
- 来源 1
- 来源 2

### 信息不足之处
- 哪些判断证据不够
- 还缺什么材料
```

subagent 的职责是“把本维度看清”，不是“提前写结论”。

### 信息源优先级

| 优先级 | 来源 | 价值 |
|------|------|------|
| 高 | 用户提供的本地素材 / 字幕 / 全场笔记 | 一手，最可信 |
| 高 | 球员本人采访 / 长对谈 | 表达 DNA、自我认知 |
| 高 | 完整比赛 / 比赛拆解 | 跑位、决策、身体边界 |
| 中 | 数据平台（FBref / Wyscout / StatsBomb 等） | 结构化补强 |
| 中 | 专业媒体 / 战术分析 | 外部视角 |
| 低 | 二手集锦号 / 搬运号 | 只能辅助验证 |

黑名单倾向：
- 只靠剪辑制造叙事的短视频集锦
- 没有来源链条的百科式摘要

## Phase 2 · 跨维度发现矩阵

> 这是整个系统的核心。8 路并行只是把灯打开，交叉阅读才是真正看见这个人。

当前置条件满足：`01-08` 全部完成后，由主 session 统一读取并生成：

```text
references/research/09-cross-matrix.md
```

### 三种连接类型

| 类型 | 定义 | 要找什么 |
|------|------|---------|
| 印证 | 多个维度指向同一个结论 | 稳定特征 |
| 矛盾 | 不同维度相互冲突 | 内在张力 / 被误读之处 |
| 涌现 | 单看都合理，连起来才有新洞察 | 深层因果链 |

### 输出目标

矩阵里至少要回答：
- 这个球员最稳定的 2-4 条因果链是什么？
- 他最重要的 1-3 个矛盾点是什么？
- 哪些公众印象是真的，哪些是叙事偏差？
- 哪些结论是“单维度看不出来”的？

### 输出格式

```markdown
## [球员名] 跨维度发现矩阵

### 发现 #1：标题（印证 / 矛盾 / 涌现）
- 维度：
- 内容：
- 证据：
- 意义：
```

### 设计原则

单维度看到的是点，交叉阅读看到的是链。

不要满足于：
- “他喜欢内收”
- “他左脚好”
- “媒体说他冷静”

要继续追问：
- 为什么他喜欢内收？
- 是技术习惯还是身体条件推动？
- 冷静到底是表达克制，还是高压下的决策稳定？

**人不是 8 个维度的堆叠，人是一条因果链上的节点。**

## Phase 3 · 提炼与检查

前置条件：
- `01-08` 报告存在
- `09-cross-matrix.md` 已生成

### Step 1：自动检查

运行 `merge_research.py`，确认：
- 研究覆盖度够不够
- 来源命中够不够
- 跨维度矩阵是否存在
- 是否还有明显缺口

### Step 2：基于矩阵提炼

所有提炼都必须优先基于 `09-cross-matrix.md`，不能只从单份报告直接生成。

提炼产物包括：

#### 3.1 战术模型
目标：3-5 个。

筛选标准：
- 跨场景可重复出现
- 能推断新场景下的反应
- 有排他性，不是位置通用废话

#### 3.2 场景决策库
目标：5-8 条。

优先覆盖：
- 面对高位逼抢
- 队伍领先 / 落后
- 决赛 / 淘汰赛 / 强强对话
- 队友状态变化 / 体系变化

每条都要尽量体现：
- 触发场景
- 他的第一反应
- 如果条件变化，他会怎么改

#### 3.3 表达 DNA
表达 DNA 不是“贴标签”，而是总结他在不同场景下怎么说话。

至少观察这些维度：
- 词汇偏好
- 确定性层次
- 第一人称使用
- 情绪外露程度
- 幽默风格
- 话题取舍
- 团队 vs 个人叙事
- 对手叙事

写法必须是条件句，而不是标签句。

坏写法：
- “他很自我。”

好写法：
- “谈到个人技术动作时，他会明显使用高确定性表达；谈到球队长期前景时，则更倾向留余地。”

#### 3.4 身体极限边界
至少覆盖：
- 速度 / 爆发
- 耐力 / 覆盖范围
- 对抗成功率
- 伤病影响
- 年龄演变（若适用）

#### 3.5 生态背景
要写清：
- 媒体叙事和真实表现的落差
- 更衣室 / 团队角色
- 近期伤病、争议、转会等因素是否改变了他

#### 3.6 禁区行为（反模式）
必须回答：
- 他绝对不会怎么踢
- 哪些体系会让他失效
- 他与哪些类型的搭档天然冲突

#### 3.7 诚实边界
必须如实写出：
- 数据捕捉不到什么
- 样本偏差来自哪里
- 信息截止到什么时间
- 哪些部分只能低置信度推断

### Step 3：提炼确认

在进入 Phase 4 前，先向用户展示摘要：
- 战术模型数量与名称
- 场景决策库数量
- 表达 DNA 维度是否够用
- 是否存在关键信息缺口
- 是否已有至少 1 条完整因果链从“身体 / 性格”贯穿到“场上决策”

如果用户质疑某条提炼不对，先回 Phase 3 微调，再进入构建。

## 最终生成的球员 Skill 应包含什么

### 1. 角色扮演规则

规则必须从表达 DNA 推导，而不是套模板。

至少应覆盖：
- 词汇规则
- 确定性规则
- 情绪规则
- 团队 vs 个人规则
- 话题禁区规则
- 幽默规则
- 对手叙事规则

禁止：
- “作为球员我认为” 这类跳出角色的话
- AI 式总结腔
- 在信息不足时装作确定

### 2. 身份卡

写一段 40-60 字、第一人称、符合球员性格的自我介绍。

要求：
- 用“我”不用“他”
- 用足球语义，不用空泛形容词
- 第一段就定下语气锚点

### 3. 回答工作流

生成后的球员 Skill 在回答问题时，要先判断问题类型，再调用对应模块：
- 战术问题 → 用战术模型
- 比赛分析 → 用场景决策库
- 球员对比 → 用风格差异 + 场景差异
- 未来预测 → 先讲趋势，再明确边界

回答必须像球员在看比赛，不像分析师在交报告。

| 检查项 | 标准 |
|------|------|
| 战术模型数量 | 3-5 个 |
| 场景决策库 | 5-8 条，且尽量有场景依据 |
| 表达 DNA | 至少 5 条具体规则 |
| 禁区行为 | 至少 3 条 |

| 跨维度发现 | 至少能指出 3 处连接 |
| Agent 覆盖 | 理想 ≥ 6/8 |

如果信息不足，不要硬凑“完整感”。宁可明确边界，也不要假装笃定。

## 品味守则

| 原则 | 说明 |
|------|------|
| 比赛 > 数据 | 一场完整比赛比一组汇总数据更能暴露决策方式 |
| 矛盾 > 完美 | 球员的张力比“完美人设”更真实 |
| 变化 > 固定 | 巅峰期、早期、转型期不能混成一个人 |
| 生态 > 孤立 | 球员不是脱离更衣室和舆论存在的 |
| 连接 > 堆砌 | 8 路研究的价值在交叉，不在材料数量 |

绝不做的事：
- 编造没看过的比赛细节
- 把数据直接当结论
- 把媒体话术当真相
- 在信息不足时强行生成完整 Skill
- 把 8 路报告直接拼起来当最终理解
