# qiuyuan-skill

> *「足球是空间的博弈，谁制造空间、谁利用空间，谁就赢。」*

**输入任意球员名，基于 Claude Code 并行调研 8 路信息 → 提炼战术框架 → 生成可运行的球员视角 Skill。**

<br>

不是复刻集锦，是提炼**他踢球的思维操作系统**。

## 效果示例

蒸馏完成后，在 Claude Code 里加载生成的球员 Skill，效果更接近下面这种 CLI 界面：

```bash
╭─ Claude Code · wulei-perspective.skill ──────────────────────────────╮
│ mode: chat   model: claude   skill: loaded                           │
╰───────────────────────────────────────────────────────────────────────╯

$ /wulei-perspective.skill

> 武磊视角已加载。
> 我是武磊。中超历史射手王，2018年金靴，2019年留洋西班牙人，攻破过巴萨球门。
> 跑位是我的语言，空间是我的战场。有什么想聊的？战术、比赛、都可以。

────────────────────────────────────────────────────────────────────────

$ 上海滩的老大是谁

> 这种说法球迷们讨论比较多。
> 我不太想争这个。在上海踢这么多年，不管是海港还是申花，都有自己的历史和球迷。足球场上看成绩，看表现，不是看谁喊得响。
> 我只管跑位、进球。其他的，让球迷去聊吧。

────────────────────────────────────────────────────────────────────────

$ 你自己不是说你们是上海滩老大嘛

> 你说的是2018年夺冠那时候吧。
> 那时候情绪确实高涨。我们拿了中超冠军，打破了恒大那么多年的垄断，27个球，金靴。那句话是在那种氛围下说的。
> 回头看，可能话说得有点满。但那时候是真的高兴，为球队高兴，为上海高兴。
> 现在我不太想聊谁是老大这种话题了。足球场上看成绩，每年都不一样。谁赢球，谁就是那个。

────────────────────────────────────────────────────────────────────────

$ 你认识申方剑嘛，他是你的忠实粉丝

> 申方剑我知道，解说员。
> 他解说的时候确实对我说过不少好话。但那些“武球王”什么的说法，解说员有他们的风格和角度，球迷也有不同的看法。
> 我自己的态度一直没变：说得好听，不如踢得实在。解说员关注我，是因为当时中国球员在欧洲踢球确实不多，大家对中国足球有期待。
> 球迷也好，解说也好，关注的是中国足球。我只是其中一部分。

────────────────────────────────────────────────────────────────────────

$ 你为什么总是丢单刀

> 这确实是我需要改进的地方。
> 单刀没进，我自己比谁都难受。但回头分析，有几个原因：
> 技术上，我习惯把球调整到右脚再射门，有时候就慢了半拍，门将已经封过来了。这是我多年的习惯，改起来不容易。
> 处理方式上，我倾向推射远角，追求角度，不是大力抽射。这种选择有时候角度没找准就没了。我知道有人说我应该更果断，这个确实在练。
> 还有身体因素，年龄上来速度没那么快了，单刀机会本来就少，处理的时候压力更大。
> 你说心态问题，我也不完全否认。有时候关键比赛，脑子里确实会想多了。但我更觉得是技术习惯和决策时机的问题，这个只能通过训练和比赛去调整。
> 进球了是运气好，队友传得好。丢单刀了是我自己没处理好。就这么简单。
```

这不是角色扮演。每个回答都基于 8 路调研提炼的**战术模型 + 表达 DNA**——捕捉的是他怎么踢球，不是他说了什么。

设计上最关键的一步不是“搜得更多”，而是 **Phase 2 的跨维度发现矩阵**：
同一条结论必须尽量同时得到技战术、生态、背景三个层面的支撑，或者明确保留矛盾，而不是把一堆材料拼贴成设定。

## 8路采集框架

球员是社会中人，也是场上决策者。采集分三层：

| 层级 | 子维度 | 说明 |
|:------|:-------|:------|
| **踢球层** | ① 空间决策 ② 技术决策库 ③ 身体与极限 ④ 体系契合 | 核心，回答他怎么踢 |
| **生态层** | ⑤ 媒体叙事 ⑥ 社交动态 | 环境，回答他处于什么处境 |
| **背景层** | ⑦ 关键时刻 ⑧ 生涯轨迹 | 背景，回答他为什么会这样踢 |

**踢球层**回答“他怎么踢的”——空间决策、技术选择、身体边界、体系统治力。  
**生态层**回答“他现在处于什么环境”——舆论标签 vs 真实表现、场下争议处理方式。  
**背景层**回答“他为什么会这样”——关键时刻表现、生涯演变轨迹。

核心设计：**Phase 2 跨维度发现矩阵**（主 session 在 8 路报告完成后统一读取并生成 `09-cross-matrix.md`）。寻找印证、矛盾、涌现，发现球员多条因果链，而非孤立事实。

## 诚实边界

**这个 Skill 能做的：**
- 用球员的战术框架分析比赛和球员对比
- 模拟他特有的说话方式和决策逻辑
- 基于身体条件推导他的踢法边界

**做不到的：**

| 维度 | 说明 |
|------|------|
| 最新动态 | 联网受限，转会/伤病等最新信息可能有缺漏 |
| 替代本人 | 他的当下状态、私下性格无法被复制 |
| 低知名度球员 | 公开信息不足时，生成的 Skill 质量会受限 |
| 完全客观 | 所有提炼都是主观判断，保留矛盾点不强制调和 |

**一个不告诉你局限在哪的 Skill，不值得信任。**

## 仓库结构

```text
qiuyuan-skill/
├── SKILL.md                  # 蒸馏流程主定义（Claude Code 触发入口）
├── README.md                 # 项目介绍与设计说明
├── LICENSE                   # MIT 许可证
└── scripts/
    ├── download_subtitles.sh # YouTube 字幕下载
    ├── srt_to_transcript.py  # 字幕清洗（SRT → 纯文本）
    ├── collect_sources.py    # 球员视频/数据元数据采集
    ├── merge_research.py     # 调研合并 + 质量检查点生成
    ├── quality_check.py      # 交付前质量自检
    └── MANUAL_COLLECTION.md  # 采集操作指南
```

注意：`references/` 不是这个仓库自带的固定目录，而是**蒸馏具体球员时**在目标 Skill 目录内生成，例如：

```text
~/.claude/skills/leao-perspective.skill/
├── SKILL.md
├── references/
│   ├── research/
│   └── sources/
└── scripts/
```

这样仓库本身只负责“方法论与工具”，具体球员产物放在目标 Skill 目录，职责更清晰。
